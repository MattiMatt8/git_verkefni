Git spurningar og svör

1. Hvað gera eftirfarandi Linux skipanir?
	cd         Change directory,breyta um folder
	cd ~       Fer í heimasvæðis folder
	ls         Sýnir skjöl/folder í folderinu sem maður er í
	pwd        Sýnir staðsetninguna manns
	mkdir      Býr til folder

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone            Vistar git repository frá öðrum stað henntugt þegar það eru margir að vinna á sama verkefni frá sama stað og er hægt að uppfæra léttilega
	git log              Sýnir öll commits af branchi og upplýsingar
	git status           Sýnir hvað hefur verið breytt á skjölum
	git diff             Borið saman tvö skjöl eða greinar
	git checkout         Skipta á milli brancha eða restore branch

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff                    Sýnir mun á því sem er verið að vinna á
	b) git diff --staged           Sýnir öll staged breytingar
	c) git diff commit1 commit2    Sýnir mun á milli tveggja commits

4. 	Hvað er útgáfustýring (e.version control)? Það tekur upp breytingar á skjölum og gefur manni stjórn á mismunandi útgáfum af verkefninu og ef eitthvað er breytt til verra þá er hægt að rúlla því til baka í fyrra version þar sem það var í lagi og heldur utanum hver breytti hverju hvenær.

5.	Hverjir eru helstu kostir við að nota GIT? Maður getur rúllað til baka ef maður hefur gert eitthvað virkilega slæmt og er gott til þess að sjá um marga sem eru að vinna á sama verkefni.  Og líka til þess að sjá um útgáfur á forriti eða svo.

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu? Á sirka 30 min fresti sérstaklega þegar maður er kominn með eitthvað sem virkar eins og virkandi version af forriti þá er best að vista það þannig ef eitthvað ruglast getur maður farið til baka.

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT?
Working directory er þar sem öll skjölin eru/geymd
Staging area þar sem maður gerir allt tilbúið áður en maður commitar
Repository þar er allt history geymt eins og öll commits o.s.frv.

8.	Hvenær er sniðugt að nota greinar (branches)? Þegar ætlað er að vinna á eitthverju nýju sem maður vill t.d. prufa áður en maður vill að almenningur sér.

9.	Hver fann upp GIT? Linus Torvalds

10.	Hvar er GIT helst notað? Það er mest notað sem hópvinnu verkefæri til að halda um versions o.s.frv.
